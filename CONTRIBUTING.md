# 贡献指南

欢迎参与 OpenClaw 新手完全指南项目的贡献！本文档说明如何参与项目，以及贡献需要遵循的规范。

## 📋 项目简介
本项目是面向新手的 OpenClaw 安装配置一站式教程，目标是降低 OpenClaw 的使用门槛，让更多用户能够快速上手部署自己的 AI 助手。所有内容遵循开源 MIT 协议。

## 🤝 贡献方式
你可以通过以下方式参与贡献：
1. **内容优化**：修正文档错误、优化表述、补充缺失内容
2. **版本更新**：跟随 OpenClaw 官方更新，同步更新教程内容
3. **功能扩展**：新增平台对接教程、部署方案、最佳实践
4. **问题反馈**：提交 Issue 报告文档错误、内容过时或用户体验问题

## 🌿 分支规范
### 分支命名规则
```
<类型>/<描述>
```
#### 类型说明：
- `feature/`：新功能/新内容开发，比如 `feature/windows-wsl-optimization`
- `fix/`：bug修复/内容修正，比如 `fix/broken-link-in-api-config`
- `docs/`：纯文档更新（指项目工程文档，非教程内容）
- `chore/`：工程配置、工具链更新等非内容变更

### 分支流程
1. 从 `main` 分支创建新的功能/修复分支
2. 在分支上完成开发
3. 提交 Pull Request 到 `main` 分支
4. 审核通过后合并到 `main` 分支，删除源分支

## 📝 Commit 信息规范
所有 Commit 必须遵循 **Conventional Commits** 规范：
```
<类型>(<范围>): <描述>
```
### 常用类型：
- `feat`：新增内容/功能
- `fix`：修复内容错误/链接失效等问题
- `docs`：更新项目工程文档（CONTRIBUTING.md/RELEASING.md等）
- `chore`：工程配置更新、依赖升级等
- `refactor`：内容重构，不影响用户阅读的结构调整
- `perf`：性能优化（针对网站/工具链）

### 示例：
```
feat: 新增Android平台安装教程
fix: 修正飞书一键安装脚本链接失效问题
docs: 更新贡献指南中Commit规范
chore: 添加CI自动检查脚本
```

## 🔍 PR 提交规范
1. **标题**：遵循 Commit 信息规范，清晰说明PR的变更内容
2. **描述**：必须包含：
   - 变更内容清单
   - 变更原因/背景
   - 检查清单（是否验证过链接、命令正确性等）
3. **变更范围**：每个PR尽量只包含单一类型的变更，避免大而全的PR
4. **审核**：PR需要至少1名维护者审核通过后才能合并

## 🎨 Markdown 格式规范
所有文档必须遵循统一的格式要求：
1. 标题层级：`#` 一级标题（页面唯一）、`##` 二级标题、`###` 三级标题，禁止跳过层级
2. 代码块：命令代码块必须指定语言（`bash`/`powershell`等）
3. 链接：使用相对路径链接内部文档，外部链接必须使用完整URL，禁止使用短链接
4. 图片：所有图片必须放在 `assets/` 目录下，使用相对路径引用，禁止引用外部图片
5. 格式：使用中文标点，英文/数字与中文之间保留空格
6. 一致性：相同功能/概念的表述必须全文档统一

## ✅ 提交前检查清单
提交PR前请确认：
- [ ] 所有链接可正常访问
- [ ] 所有命令可正确执行
- [ ] 内容符合 OpenClaw 官方最新文档
- [ ] 格式符合 Markdown 规范
- [ ] 没有错别字和语法错误

## 🚀 审核与合并
1. PR提交后会自动触发CI检查（链接有效性、格式检查等）
2. CI全绿后维护者会进行内容审核
3. 审核通过后会合并到 `main` 分支，自动发布到线上
4. 贡献者会被加入到贡献者列表中

感谢你的贡献！💖
