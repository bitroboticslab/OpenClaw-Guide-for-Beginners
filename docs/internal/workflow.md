# 项目工作流规范

本文档定义本项目的标准工作流程，所有贡献和维护操作都需要遵循此流程，保证项目的一致性和质量。

## 🎯 完整工作流程

```
调研搜索 → 需求明确 → 撰写执行计划 → 按计划执行 → 分支提交 → 版本维护 → 新建PR → CI全绿修复 → 审核合并
```

---

### 1. 调研搜索阶段
**目标**：收集足够的信息，保证内容的准确性和及时性。
**操作步骤：**
1. 检查OpenClaw官方最新文档、Release Notes、变更日志
2. 搜索相关教程、社区讨论、用户反馈，了解常见问题
3. 验证所有命令、链接、配置的正确性
4. 调研同类教程的优点和不足，确定优化方向
**输出：** 调研笔记、问题列表、优化方向

---

### 2. 需求明确阶段
**目标**：清晰定义本次变更的范围和目标，避免范围蔓延。
**操作步骤：**
1. 明确本次变更的背景和原因
2. 列出需要完成的具体任务清单
3. 确定变更的优先级和时间要求
4. 评估变更的影响范围，避免对其他内容造成破坏
**输出：** 需求清单、任务分解表

---

### 3. 撰写执行计划阶段
**目标**：制定可执行的实施计划，保证开发过程有序进行。
**操作步骤：**
1. 将任务分解为可独立执行的小步骤
2. 为每个步骤分配时间和优先级
3. 确定需要用到的工具和资源
4. 制定检查点和验收标准
**输出：** 执行计划文档、验收标准

---

### 4. 按计划执行阶段
**目标**：按照计划完成内容开发，保证质量。
**操作步骤：**
1. 从main分支创建新的功能/修复分支
2. 按照执行计划逐步完成任务
3. 每完成一个小步骤就进行本地验证
4. 及时记录遇到的问题和解决方案
5. 定期同步main分支的最新变更，避免冲突
**输出：** 完成的内容变更、问题记录

---

### 5. 分支提交阶段
**目标**：按照规范提交代码（文档）变更，保证可追溯性。
**操作步骤：**
1. 检查变更内容，确保只包含本次计划的变更
2. 按照Commit规范编写提交信息
3. 本地运行CI检查脚本，保证没有低级错误
4. 推送分支到远程仓库
**输出：** 提交完成的分支、本地检查报告

---

### 6. 版本维护阶段
**目标**：保证版本信息的一致性和可追溯性。
**操作步骤：**
1. 更新CHANGELOG.md，添加本次变更的记录
2. 检查所有文档中的版本号是否正确
3. 如果是大版本更新，更新版本号相关的所有引用
4. 验证所有链接和命令的正确性
**输出：** 完成版本更新的文档

---

### 7. 新建PR阶段
**目标**：创建符合规范的PR，便于审核。
**操作步骤：**
1. 按照PR规范填写标题和描述
2. 在描述中列出所有变更内容、变更原因、检查清单
3. 关联相关的Issue（如果有）
4. 指派审核人
**输出：** 创建完成的PR

---

### 8. CI全绿修复阶段
**目标**：保证所有自动化检查通过。
**操作步骤：**
1. 查看CI运行结果，定位失败原因
2. 修复问题，提交新的Commit
3. 重复直到所有CI检查全绿
4. 通知审核人进行内容审核
**输出：** CI全绿的PR，等待审核

---

### 9. 审核合并阶段
**目标**：保证内容质量符合要求。
**操作步骤：**
1. 审核人进行内容审核，提出修改意见
2. 开发者根据意见进行修改，直到审核通过
3. 审核通过后合并到main分支
4. 删除源分支
**输出：** 合并到main分支的变更，本次任务完成

---

## 🔧 工具链支持
### 自动化脚本
- `scripts/check-openclaw-update.sh`：自动检测OpenClaw版本更新，创建PR
- `scripts/ci-check.sh`：CI检查脚本，检查链接有效性、格式正确性、命令可执行性
### Cron任务配置
**版本自动检测任务**：每周一早上8点运行，检查OpenClaw官方更新
```bash
# 添加cron任务
crontab -e
# 添加以下内容
0 8 * * 1 /bin/bash /root/.hermes/projects/OpenClaw-Guide-for-Beginners/scripts/check-openclaw-update.sh >> /var/log/openclaw-update.log 2>&1
```
### 质量检查
- 链接检查：自动检查所有内外链接是否有效
- 格式检查：自动检查Markdown格式是否符合规范
- 拼写检查：自动检查常见错别字

---

## ✅ 质量标准
所有合并到main分支的内容必须满足：
1. 所有链接可正常访问
2. 所有命令可正确执行
3. 内容符合OpenClaw官方最新文档
4. 格式符合Markdown规范
5. 没有错别字和语法错误
6. 表述清晰易懂，适合新手阅读
7. 所有安全提示到位，避免用户操作风险

---

本工作流会根据实际运行情况持续优化，如有改进建议可以提交Issue讨论。
