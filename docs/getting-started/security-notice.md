# 安全运行重要提示
## ⚠️ Windows用户安全必读
### 风险提示
直接在Windows本地安装OpenClaw存在以下安全风险：
1. **误操作风险**：OpenClaw拥有文件系统访问权限，插件或Agent执行的错误操作可能导致重要文件被删除或修改
2. **系统级风险**：如果以管理员权限运行，恶意插件或错误指令可能对系统造成不可逆的损坏
3. **兼容性风险**：Windows平台的文件系统权限模型与Linux差异较大，可能出现预期外的文件操作行为
### 推荐方案（新手必选）
✅ **强烈推荐使用WSL（Windows子系统Linux）安装**
WSL模式的安全优势：
- 运行在独立的Linux沙箱环境中，与Windows主系统隔离，即使出现误操作也不会影响Windows系统文件
- 权限控制更严格，默认无法直接访问Windows系统的敏感目录
- 兼容性更好，与生产环境运行模式一致，后续维护更方便
- 性能与原生Linux基本一致，没有额外 overhead
### 本地安装注意事项（仅适用于有经验用户）
如果你坚持使用Windows本地安装，请务必遵循以下安全规则：
1. **不要以管理员权限运行**：仅使用普通用户权限启动OpenClaw
2. **开启沙箱模式**：在配置文件中开启`security.sandbox.enabled = true`，限制文件系统访问范围
3. **配置目录白名单**：仅允许OpenClaw访问必要的工作目录，禁止访问系统目录、文档目录等敏感位置
4. **定期备份**：重要文件提前备份，避免意外损失
## 🔒 通用安全最佳实践
所有平台用户都应该遵循以下安全规则：
1. **最小权限原则**：不要以root/管理员权限运行OpenClaw，使用普通用户权限即可
2. **插件来源可信**：仅安装来自官方仓库或可信来源的插件，不要随意安装第三方未知插件
3. **敏感信息隔离**：不要在配置文件中明文存储密码、API密钥等敏感信息，使用环境变量或秘钥管理工具
4. **定期更新**：及时更新到最新版本，修复安全漏洞
5. **网络安全**：如果对外暴露服务，务必配置身份认证和HTTPS加密，不要直接对公网开放
## 🚨 紧急处理
如果出现误操作导致文件被删除或系统损坏：
1. 立即停止OpenClaw服务：`openclaw gateway stop`
2. 不要向被删除文件所在分区写入新数据，避免覆盖
3. 使用数据恢复工具尝试恢复文件
4. 提交Issue告知我们具体情况，我们会协助排查问题原因
